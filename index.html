Processing

In CSS, the page is processed from top to bottom. That means that CSS on the bottom of the page overrides CSS at the bottom of the page.

Selectors

When CSS elements are laid out in a CSS stylesheet with only spaces between them, the CSS rule applies to nested elements in that specific order.

If CSS classes or IDs are directly next to eachother, that means both are required for the rule to be enacted. I am not sure if normal html elements work the same, but I assume not.

Margin/Padding

For Margin or Padding property in CSS, the order is Top/Bottom Right/Left, or Top, Right, Bottom, Left with no commas. Only spaces separate these values.

Clear: Both

Clear both will ignore floats form both sides and place itself below those elements.

Clearfix

The following class may be applied to make the size of the parent element the same as the children

clearfix::after{
clear:both;
content: "";
display:table;}

There are some apparent stipulations that I have not looked into that will cause scrollbars to appear if improper margin and padding settings are employed.

Box-shadow

Box-shadow properties operate as the following:

x-offset:

and

y-offset:

will adjust shadow in x and y directions. Other options include blur, color, and spread. Color changes color, blur increases blur, and spread increases size.

Media Queries

While media queries commonly check for screen size or resolution, they can also check for other things such as orientation (landscape or portrait), or if the document is being printed.

Initial value

If the initial value is used for a rule, it will set the value to the previous, initial value. If there is no initial value that was set by the user, the initial value of the document will be set. This may be useful for font sizes but I'm not sure.

Grid

CSS grid has revolutionized the CSS landscape. Something like

grid-template column: 200px 200px 200px;

will create three columns of 200px. New unit fr, which stands for fraction, will allocate equally for number of fractions available. A repeat function exists

grid-template-columns:repeat(3, 1fr);

will give the same output as

grid-template-column: 1fr 1fr 1fr;

Repeat can be used with other values and grid-template-column creates implicit rows. That means one row.

grid-auto-rows:

will take values for size of rows.

grid-justify-items:

Grid by default takes into account borders in an elements sizing, which is super convenient in my experience. Multidirectional floats?

First Class Functions

Functions are treated as any variable would be and can be passed as arguments

Event Driven Environment

Listeners that watch for specific events of different types make things happen and are part of what drives Javascript at its core.

Only the event that is being listened for will load when triggered, not the whole page. This leads to faster load speeds

Closure

Javascript notices use of a variable outside of the scope of an event, and will save the variable for further use. For example If a varialbe has been saved in memory due to use in an event/function, it is a closure.

.removeEventListener

will take a variable out of closure. This is useful due to memory leaks, if a memory leak happens the user experience may be fucked.

Scope

Scope is where variables are able to be accessed.

Context

Context is the value of javascript tool this

The root scope in HTML is the window object.

If var is created in scope of a child function it will be undefined in a parent function call, however children have access to parent variables

Scope conflict is defining a variable in child scope that already exists in a parent scope.

Child function looks for variable within its own scope before looking through parent element. If it looks through parent element regardless, I do not know.

Polluting the root scope is forgetting to define a variable. Strict mode will help check for pollution.

In the root scope, this is the window.

If a function is run from the window, this will still be in the window because that is what it was run in.

There are three ways to change context

  • Call
  • Apply
  • Bind

object = {foo:function(){console.log(this)

window.object=obj is this

obj.foo.call(window)=window is this

Giving call more than one argument is passing arguments to the function.

Apply is the same as call, except it takes two arguments and the second argument is an array.

Bind doesn't apply context. Bind binds a function to an ojbect. For example binding an obj to window will always return that object within the context window

find will search within context of this (?)

    Push

    Push will upload to Git

    The following are directions to push into a repository
    First, initialize in the correct directory.

    git init

    Then, add the new files. This command adds them all:

    git add .

    Next, create a commit and add a description:

    git commit -m "DESCRIPTION"

    Use

    git remote add origin REMOTE-REPOSITORY-URL

    then

    git remote -v

    to add your own git url that you are uploading to, then verify it.

    Finally, push it to your repository

    git push origin master

    Some of these steps are not necessary if you already did the steps in this section before.
